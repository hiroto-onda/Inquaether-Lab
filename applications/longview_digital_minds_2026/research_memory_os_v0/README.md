# Research Memory OS v0
Status: Manual Evidence-Gathering Support

## Purpose

Research Memory OS v0 is a manual support folder for collecting source references and evidence notes for the Longview Digital Minds 2026 application.

It exists to help organize observed material before application drafting. It is not an automation system, research sync process, or source repository maintenance workflow.

## Scope

This v0 scope is limited to:

- Recording source references.
- Capturing short evidence notes.
- Grouping evidence by Longview application section.
- Tracking open questions for later human review.
- Keeping all work inside `applications/longview_digital_minds_2026/research_memory_os_v0/`.

This v0 scope excludes:

- Automation.
- Scripts.
- API usage.
- Scheduled tasks.
- Background sync.
- Repository edits outside this application support folder.

## Cost Limit

The cost limit is under 3,000 JPY per month.

For v0, the expected cost is 0 JPY because the workflow is manual and does not use APIs, paid services, scripts, scheduled jobs, or automated processing.

## Read-Only Boundary

All source material must be treated as read-only.

Do not edit:

- `ci_theory`
- `lumenia-exogram`
- Existing `ci-memory-weekly-sync` automation
- `index.html`
- Repository-level `README.md`
- `docs/`
- GitHub Pages settings

Source repositories and external materials may be reviewed only to record references and evidence notes in this v0 folder.

## Relationship to Longview Application

This folder supports the Longview Digital Minds application by keeping application evidence organized before drafting.

The intended output is a clearer set of references for:

- Motivation evidence.
- Research evidence.
- Fellowship plan evidence.
- Future career evidence.
- Funding need evidence.
- Website and GitHub evidence.
- Open questions.
