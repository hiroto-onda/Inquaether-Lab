# Research Memory OS v0 Workflow
Status: Manual Workflow Only

This workflow supports evidence gathering for the Longview Digital Minds application. It does not use automation, scripts, APIs, scheduled tasks, or source repository edits.

## Step 1: Select Small Source Set

Choose a small source set for one application section or one question.

Examples:

- One repository page or document.
- One Google Drive file.
- One local note draft.
- One ChatGPT export segment.

Do not scan broad archives or run automated ingestion.

## Step 2: Record Source Metadata

Add the source to `source_manifest.md`.

Record:

- Source name.
- Location or reference.
- Application relevance.
- Review status.
- Short notes about sensitivity, uncertainty, or exclusions.

## Step 3: Extract Evidence Notes

Read the selected source manually and extract only concise evidence notes.

Each note should include:

- What was observed.
- Where it came from.
- Why it may matter for the Longview application.
- Whether it needs confirmation before reuse.

Do not paste long excerpts or unsupported interpretations.

## Step 4: Group by Application Section

Add each evidence note to the matching section in `evidence_index.md`:

- Motivation Evidence.
- Research Evidence.
- Fellowship Plan Evidence.
- Future Career Evidence.
- Funding Need Evidence.
- Website and GitHub Evidence.
- Open Questions.

If a note could fit multiple sections, choose the strongest section and mention cross-use in the application-use field.

## Step 5: Human Review

Review each evidence note before using it in an application draft.

Check:

- Is the source appropriate to cite or summarize?
- Is the note accurate?
- Is the claim too strong for the available evidence?
- Is any sensitive information included?
- Does the note need a better source?

## Step 6: Reuse in Application Drafts

Only after review, reuse evidence notes in application drafts.

When reusing evidence:

- Keep the original source reference nearby.
- Separate evidence from interpretation.
- Avoid overstating claims.
- Preserve uncertainty when needed.
- Do not edit source repositories or existing automations.
